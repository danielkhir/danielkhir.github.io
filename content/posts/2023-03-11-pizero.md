## Intro

During a free weekend, I used Ansible to automatically configure my Pi Zero W to block ads with Pi-hole.

I bought a Raspberry Pi Zero W for a very reasonable 20€. The Pi Zero boasts a 1GHz single-core ARMv6 CPU, 512MB of RAM, and is powered by Micro-USB. It also has Bluetooth 4.0 and WLAN capabilities.

I wanted to accomplish two things with the Pi:
- increase my familiarity with Linux by setting up Pi-hole on the machine.
- automate logging into my local network, as the place I'm currently living in requires a sign up every 24 hours.
- dive into Infrastructure as Code with Ansible and Vagrant.

As it arrived on a Friday, I spent the weekend tinkering with it. This post details my experience setting it up.

## Initial Setup

Along with the Pi itself, I got a few other peripherals, such as a 32GB SD card for storage and a power adapter.

The first step was none other than to install an OS on the Pi, and configure it for a headless setup with SSH so that I can access it without it being connected to a monitor.

### Steps

1. Flash the [Raspberry Pi OS Lite](https://downloads.raspberrypi.org/raspios_lite_armhf/images/) image with the [Raspberry Pi Imager](https://downloads.raspberrypi.org/imager/).

2. Navigate to the Pi's `/boot` folder and create the following files:

    - `ssh` - Enables SSH access.
    - `wpa_supplicant.conf` - Defines WiFi connection details. An example can be found [here](https://www.raspberrypi.com/documentation/computers/configuration.html#setting-up-a-headless-raspberry-pi).
    - `userconf.txt` - Define user in `user:password` format. The password should be encrypted with `openssl passwd -6`.

3. Generate a SSH key and copy the public key into the Pi's `.ssh/authorized_keys`. (I skipped this step by accident and got locked out, oops...)

    ```bash
    ssh-keygen -t ed25519 -C "pizero"
    ssh-copy-id -i key.pub user@pizero
    ```

4. Configure secure SSH by disabling password access. Change the following lines in `/etc/ssh/sshd_config`, then restart sshd.

    ```bash
    ChallengeResponseAuthentication no
    PasswordAuthentication no
    UsePAM no

    # Restart SSH service
    sudo service ssh reload
    ```

## Pi-hole

[Pi-hole](https://docs.pi-hole.net/) is a network-wide ad-blocker, which can also function as a local DHCP server. I installed it on bare metal. The install process was relatively straightforward, as the one-step install script guides you through the necessary steps.

After finishing the installation, I assigned a static IP for the Pi on my router.

## Network Autologin

Manually logging in every 24 hours into the local network is a hassle. It interrupts whatever I'm doing at the time, and I waste precious moments trying to reconnect.

The solution - a simple Python script that checks if I'm logged in by pinging Google's DNS servers.

If no response was received, it implicitly assumes that the network session has ended and a login is required. Subsequently, a login is performed by sending a GET request to the network gateway. This login attempt is then logged in a CSV file.

Finally, I created two cron jobs to automate the process: one to run it every minute, another to clean the contents of the CSV every month to prevent it from getting too lengthy. The jobs look like this:

```plaintext
* * * * * python3 /path/to/login.py
0 0 * * 1 rm /path/to/login_log.csv
```

## IaC with Ansible

With [Ansible](https://www.ansible.com/), I can define the steps to configure the Pi, such as configuring SSH and setting up cronjobs, in so-called playbooks and apply them automatically. Ansible checks the Pi to see if its current state matches that in the playbook.

Playbooks are also a form of documentation and can be version-controlled. If something goes wrong and the Pi needs a clean install, running the playbook will restore all the configuration without manual intervention. Neat, huh?

As this was my first time working with Ansible, I needed a playground to test out what I've written in the playbook. I couldn't use the Pi, as that was already set up and I didn't want to intefere with it any further.

The solution was [Vagrant](https://developer.hashicorp.com/vagrant/tutorials/getting-started), a virtual machine provisioner by Hashicorp, which lets me spin up and destroy VMs with a few commands.

I can then test the playbook on a few throwaway Vagrant VMs to confirm that everything was working as intended.

## Conclusion

This was definitely a fun weekend project. I learnt how to:

- configure SSH
- manage cronjobs with crontab
- use Ansible and Vagrant

In the weeks since implementing this, the network autologin alone has been well worth the effort. I look forward to messing around with the Pi in future projects.
